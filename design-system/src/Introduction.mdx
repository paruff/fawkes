# Fawkes Design System

Welcome to the Fawkes Design System! This is a comprehensive collection of reusable components, design tokens, and guidelines for building consistent user interfaces across the Fawkes platform.

## üéØ Purpose

The Fawkes Design System provides:

- **41+ Production-Ready Components** - Covering layout, typography, forms, feedback, navigation, display, and data
- **Design Tokens** - Centralized color, typography, spacing, and other design decisions
- **Accessibility** - WCAG 2.1 AA compliant components with full ARIA support
- **TypeScript Support** - Full type safety and IntelliSense
- **Comprehensive Documentation** - Examples, API documentation, and best practices

### Component Categories

- **Layout** (5 components): Container, Grid, Stack, Spacer, Box
- **Typography** (3 components): Heading, Text, Code
- **Forms** (13 components): Button, IconButton, ButtonGroup, Input, Select, Checkbox, Radio, Switch, Textarea, FormField, FormLabel, FormHelperText, FormErrorMessage
- **Feedback** (6 components): Alert, Toast, Spinner, Progress, Badge, Skeleton
- **Navigation** (5 components): Tabs, Breadcrumbs, Pagination, Menu, Link
- **Display** (8 components): Card, Avatar, Chip, Tooltip, Modal, Drawer, Divider, Image
- **Data** (2 components): Table, List

## üé® Design Tokens

All components use design tokens for consistent styling:

```typescript
import { tokens } from '@fawkes/design-system';

// Colors
const primaryColor = tokens.colors.primary[500];

// Typography
const headingSize = tokens.typography.fontSize['2xl'];

// Spacing
const marginLarge = tokens.spacing[8];
```

## ‚ôø Accessibility

Every component in the design system follows accessibility best practices:

- Semantic HTML elements
- ARIA attributes where needed
- Keyboard navigation support
- Screen reader support
- Focus management
- Color contrast compliance (WCAG 2.1 AA)
