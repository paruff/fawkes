üîß Common Issues & Solutions

Q: My deployment is stuck in "Building" status

Solution:

  1. Check build logs in Fawkes UI
  2. Common causes:
    • Missing dependencies in package.json/requirements.txt
    • Build script errors
    • Dockerfile issues
  3. Run fawkes logs build to see detailed errors
Q: Application crashes immediately after deployment

Solution:

  1. Check logs: fawkes logs --tail 100
  2. Common causes:
    • Missing environment variables/secrets
    • Database connection failures
    • Port configuration (must listen on PORT env var)
  3. Test locally: docker build -t test . && docker run -p 8080:8080 test
Q: How do I rollback a bad deployment?

Solution:

# List recent deployments
fawkes deployments list

# Rollback to previous version
fawkes rollback

# Or rollback to specific version
fawkes rollback --version v1.2.3
Q: I can't access my application URL

Solution:

  1. Verify deployment is healthy: fawkes status
  2. Check if pods are running: fawkes pods
  3. Are you on VPN? (May be required for internal apps)
  4. Try: fawkes open to get correct URL
  5. Check ingress: fawkes ingress status
Q: How do I add environment variables/secrets?

Solution:

# Add secret via CLI
fawkes secret set DATABASE_URL "postgres://..."

# Or add to .fawkes/config.yaml
secrets:
  - DATABASE_URL
  - API_KEY

# Then set values in Fawkes UI > Secrets tab
Q: Deployment succeeded but app shows 502/504 errors

Solution:

  1. Health check may be failing. Ensure your app:
    • Listens on the correct port (from PORT env var)
    • Responds to /health or / endpoint
    • Starts within 60 seconds
  2. Check: fawkes health
  3. View health check logs: fawkes logs --health

üìö Git Basics for Beginners

Git is a version control system that tracks changes to your code.

Essential Git Commands

# Clone a repository (download code)
git clone <url>

# See what files changed
git status

# Stage changes for commit
git add .
# (The . means "all files")

# Commit changes with a message
git commit -m "Your message here"

# Push to GitHub (upload your changes)
git push origin main

# Pull latest changes (download updates)
git pull

Good Commit Messages

Write clear commit messages that explain WHAT and WHY:

  • ‚úÖ Good: "Fix login bug where users couldn't reset password"
  • ‚úÖ Good: "Add user profile page with avatar upload"
  • ‚ùå Bad: "Update code"
  • ‚ùå Bad: "fix"
