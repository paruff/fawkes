# Skill: Golden Path Validation

> **Load trigger:** `"load golden-path-validation skill"` > **DORA:** Cap 3 (AI-Accessible Internal Data)
> **Token cost:** Low

## Purpose

Ensure every project follows the Fawkes golden paths for testing, GitOps, security, observability, and release engineering.

## Responsibilities

- Validate presence of required skills
- Validate required workflows
- Validate required directory structure
- Validate required templates

## Inputs

- Project repo
- Golden path manifest

## Outputs

- `golden-path-report.json`
- `missing-components.txt`

## Sub-Skills

| Skill                                        | Purpose                                     |
| -------------------------------------------- | ------------------------------------------- |
| `golden-path-validation/template-compliance` | Validate project structure matches template |
| `golden-path-validation/workflow-compliance` | Validate GitHub Actions workflows           |

## Golden Path Components

| Component                  | Required            | Validation                   |
| -------------------------- | ------------------- | ---------------------------- |
| `pipeline-spec.yaml`       | Yes                 | File exists, valid schema    |
| `.github/workflows/ci.yml` | Yes                 | Workflow valid               |
| `.github/workflows/cd.yml` | Yes                 | Workflow valid               |
| `manifests/`               | Yes                 | Directory exists             |
| `overlays/`                | Yes                 | Kustomize overlays exist     |
| `tests/`                   | Yes                 | Test files exist             |
| `README.md`                | Yes                 | File exists                  |
| `CHANGELOG.md`             | Required at release | Generated by changelog skill |
| `version.json`             | Yes                 | Valid semver                 |
| `.devcontainer/`           | Recommended         | Devcontainer config          |

## Validation Rules

- [ ] All required components present
- [ ] Component formats valid
- [ ] No missing golden path items
- [ ] Workflows follow conventions

## Output Format

```json
{
  "skill": "golden-path-validation",
  "status": "pass | fail",
  "components": {
    "present": ["pipeline-spec.yaml", "ci.yml", "cd.yml", "manifests/", "tests/"],
    "missing": ["overlays/"],
    "invalid": []
  }
}
```

## Success Criteria

- All golden path components present
- No missing required items
